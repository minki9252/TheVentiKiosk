# ☕ TheVentiKiosk

> Qt/C++로 구현한 **더벤티 키오스크 주문 시뮬레이션 프로젝트**입니다.  
> 메뉴 탐색부터 옵션 선택, 장바구니, 쿠폰, 포인트 적립, 카드 결제, 영수증 선택, 주문번호 발급까지 실제 카페 키오스크의 주문 흐름을 단계별 화면으로 구성했습니다.

<br>

![TheVentiKiosk Logo](VentiApp/Image/Benner/logo.png)

## 📌 프로젝트 소개

**TheVentiKiosk**는 카페 무인 주문 환경을 학습 목적으로 재현한 데스크톱 GUI 애플리케이션입니다.  
사용자는 매장/포장 여부를 선택한 뒤 카테고리별 메뉴를 확인하고, 음료 옵션을 고른 후 장바구니에 담아 결제 과정을 진행할 수 있습니다.

프로젝트는 UI 애플리케이션 영역인 `VentiApp`과 데이터베이스 및 메뉴 데이터 처리를 담당하는 `VentiCore`로 분리되어 있습니다.

## ✨ 주요 기능

### 🛒 주문 흐름

- 매장 / 포장 주문 선택
- 신메뉴, 커피, 버블티&티, 블렌디드, 베버리지, 에이드, 디저트 카테고리 제공
- 메뉴 이미지, 이름, 가격 표시
- 선택한 메뉴를 옵션 적용 후 장바구니에 추가
- 장바구니 수량 증가 / 감소, 개별 삭제, 전체 비우기 지원
- 총 결제 금액 실시간 계산

### 🧋 메뉴 옵션

- SQLite에 저장된 옵션 그룹을 기반으로 옵션 버튼 동적 생성
- 단일 선택 / 다중 선택 옵션 그룹 처리
- 사이즈, 원두, 샷, 시럽, 탄산수 변경, 토핑 등 옵션 구성
- 옵션 추가 금액을 반영한 최종 메뉴 금액 계산
- 선택 옵션 요약 표시

### 🎟️ 쿠폰 & 포인트

- 쿠폰 적용 여부 선택
- 쿠폰 번호 입력 및 결과 화면 전환
- 할인 금액 적용 후 결제 금액 갱신
- 전화번호 입력 기반 포인트 적립 플로우
- 주문 수량 기준 적립 스탬프 계산
- 개인정보 보호를 위한 전화번호 일부 마스킹 표시

### 💳 결제 시뮬레이션

- 카드 결제 방식 선택
- 카드 삽입 대기 화면
- 30초 카운트다운 처리
- 카드 삽입 후 로딩 화면 표시
- 결제 완료 / 취소 / 시간 초과 흐름 처리

### 🧾 영수증 & 주문번호

- 영수증 출력 여부 선택
- 주문번호 자동 발급
- 주문 완료 화면 표시
- 일정 시간 후 자동 종료 처리

## 🧩 데이터베이스 구성

앱 실행 시 SQLite DB를 초기화하고 주요 테이블을 생성합니다.

| 테이블 | 역할 |
| --- | --- |
| `CATEGORIES` | 메뉴 카테고리 정보 |
| `MENU_INFO` | 메뉴명, 가격, 이미지 경로, 품절 여부 |
| `OPTION_GROUPS` | 옵션 그룹명 및 단일/다중 선택 여부 |
| `OPTIONS` | 옵션명 및 추가 금액 |
| `MEMBERS` | 회원 전화번호, 스탬프, 누적 결제 금액 |
| `ORDERS` | 주문 정보 |
| `ORDER_ITEMS` | 주문 상세 항목 |

## 🚀 실행 방법

### Qt Creator

1. Qt Creator에서 `TheVentiKiosk.pro` 파일을 엽니다.
2. 사용 가능한 Qt Kit를 선택합니다.
3. 전체 프로젝트를 빌드합니다.
4. `VentiApp`을 실행합니다.

### qmake

```bash
qmake TheVentiKiosk.pro
make
```

Windows 환경에서는 Qt Kit에 따라 아래 명령을 사용할 수 있습니다.

```bash
mingw32-make
```

또는

```bash
nmake
```

## 💡 구현 포인트

- **Core/App 분리**  
  DB 초기화, 테이블 생성, 메뉴/옵션 조회 로직은 `VentiCore`에서 관리하고, UI 흐름은 `VentiApp`에서 처리하도록 역할을 분리했습니다.

- **JSON 기반 메뉴 초기화**  
  `menus.json`의 카테고리와 메뉴 데이터를 읽어 SQLite에 초기 적재하고, UI는 DB 조회 결과를 기반으로 메뉴를 표시합니다.

- **동적 옵션 UI 생성**  
  옵션 그룹과 옵션 목록을 DB에서 가져와 버튼을 동적으로 생성합니다. 옵션 선택 상태에 따라 추가 금액과 최종 금액을 즉시 갱신합니다.

- **Signal/Slot 기반 화면 전환**  
  Qt의 signal/slot 구조를 활용해 메뉴 선택, 장바구니 추가, 결제 단계 이동, 영수증 완료 등의 흐름을 연결했습니다.

- **키오스크 결제 경험 시뮬레이션**  
  실제 PG 연동 대신 카드 선택, 카드 삽입 대기, 로딩, 완료/취소/시간 초과 상태를 화면 전환과 타이머로 구현했습니다.
